## Next steps for Blitz

### Language features (tinyc)
- switch/case
- enum support
- Float globals and struct fields

### Backend optimization
- Param precoloring optimization: currently skips ALL params when block has calls; could check per-param liveness at each call point
- Loop optimizations (LICM, strength reduction, induction variables)
- Better register coalescing across blocks
- Better spill code: split live ranges at loop boundaries, register hints
- Tail call optimization
- Block param const propagation for multi-predecessor case

### E-graph optimization (from docs/egraph-optimization-roadmap.md)
- Shift+mask optimization: And(Shr(a,n), mask) when shift zeroes masked bits
- Type-width analysis: (min_bits, signed) per class; enables narrowing, avoids unnecessary extensions
